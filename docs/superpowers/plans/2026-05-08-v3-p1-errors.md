# v3 Phase 1 — Error foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace v2's catch-all `Error::ParseFailed(Box<dyn Error>)` plus `From<&str>` / `From<String>` "stringly-typed" error funnel with the structured v3 error model from spec §3.2: `Error` (`Io` / `UnsupportedFormat` / `ExifNotFound` / `TrackNotFound` / `Malformed { kind, message }` / `UnexpectedEof { context }`), public `EntryError` (`Truncated` / `InvalidShape` / `InvalidValue`), `ConvertError` (peer-level), and `MalformedKind`.

**Architecture:** New error types live in `src/error.rs` (extended). Internal `ParseEntryError` is replaced by the new public `EntryError`. `From<EntryError> for Error` routes to `Error::Malformed { kind: IfdEntry, .. }`. `ConvertError` is a peer of `Error` (no `From` between them). Migration is performed in additive-then-subtractive order so each commit compiles: add new variants, migrate sites one category at a time, delete the old variants last.

**Tech Stack:** `thiserror` 2.0 (already a dep). MSRV bumps to 1.83 in this phase to unlock `Option::is_some_and` and the `expect` lint policy used by later phases.

**Spec sections covered:** §3.2 (full), §5.2 (migration guide entries for errors).

**Out of scope:** anything in §3.3+ — no `MediaSource` / `MediaParser` / `EntryValue` / `ExifTag` API changes here. P1 is a pure error-shape cutover.

```rust
/// Categorizes the format-family that produced a `Error::Malformed`.
///
/// Used by downstream code to react differently depending on which container
/// or sub-structure failed to parse — e.g. a corrupted JPEG segment is
/// recoverable for many use cases while a corrupted EBML element typically
/// is not.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
#[non_exhaustive]
pub enum MalformedKind {
    JpegSegment,
    TiffHeader,
    IfdEntry,
    IsoBmffBox,
    EbmlElement,
    Cr3Container,
    Heif,
    Raf,
}

impl std::fmt::Display for MalformedKind {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        let s = match self {
            Self::JpegSegment => "jpeg segment",
            Self::TiffHeader => "tiff header",
            Self::IfdEntry => "ifd entry",
            Self::IsoBmffBox => "iso-bmff box",
            Self::EbmlElement => "ebml element",
            Self::Cr3Container => "cr3 container",
            Self::Heif => "heif",
            Self::Raf => "raf",
        };
        f.write_str(s)
    }
}
```

```rust
/// Errors from conversions that are *orthogonal* to file parsing: parsing a tag
/// name from a string, narrowing an `IRational` into a `URational`, building a
/// `LatLng` from decimal degrees, parsing an ISO 6709 coordinate string.
///

/// for Error`. Downstream code that needs to combine file-level errors and
/// conversion errors should define its own wrapper enum (the standard

pub enum ConvertError {
    #[error("unknown ExifTag name: {0}")]
    UnknownTagName(String),

    #[error("invalid ISO 6709 coordinate: {0}")]
    InvalidIso6709(String),

    #[error("rational has negative value")]
    NegativeRational,

    #[error("decimal degrees out of range or non-finite: {0}")]
    InvalidDecimalDegrees(f64),
}
```

Append to `src/error.rs`:

```rust
/// Per-entry parsing error inside an IFD. Three variants cover every
/// entry-level failure mode actually seen in the wild; the type is
/// deliberately narrow and structured (no `String` payload) so downstream
/// can `match` on the cause.
#[derive(Debug, Clone, thiserror::Error)]
#[non_exhaustive]
pub enum EntryError {
    /// Entry header declared more bytes than the IFD/file actually has.
    #[error("entry truncated: needed {needed} bytes, only {available} available")]
    Truncated { needed: usize, available: usize },

    /// `(format, count)` is incompatible with the tag, or `format` is unknown.
    #[error("invalid entry shape: format={format}, count={count}")]
    InvalidShape { format: u16, count: u32 },

    /// Bytes parsed successfully but the decoded value is invalid for this tag
    /// (non-UTF-8 string, malformed datetime, GPS ref outside `N/S/E/W`, ...).
    /// `&'static str` (not `String`) because the failure modes are a finite,
    /// fixed set.
    #[error("invalid value: {0}")]
    InvalidValue(&'static str),
}

/// `EntryError` propagates upward as `Error::Malformed { kind: IfdEntry, .. }`,
/// letting downstream code use `?` to bubble per-entry errors to file-level.
impl From<EntryError> for Error {
    fn from(e: EntryError) -> Self {
        Error::Malformed {
            kind: MalformedKind::IfdEntry,
            message: e.to_string(),
        }
    }
}
```

- [ ] **Step 4.6: Map every internal `ParseEntryError` construction site**

This is mechanical but tedious. The mapping rules are:

| Old construction | New construction | Rationale |
|------------------|------------------|-----------|
| `ParseEntryError::EntrySizeTooBig` | `EntryError::Truncated { needed, available }` — at the call site, you have the requested entry size and the buffer length. Pass both. | EntrySizeTooBig is exactly the truncation case. |
| `ParseEntryError::Unsupported(s)` | `EntryError::InvalidShape { format, count }` — the call site has `data_format` and `components_num` in scope. | Unsupported always means "we don't know how to decode (format, count)". |
| `ParseEntryError::InvalidData(s)` where `s` describes a value-decode failure (UTF-8, datetime, GPS ref) | `EntryError::InvalidValue("<static-str>")` — pick the static string from the table below. | These are decode-time failures, not shape failures. |
| `ParseEntryError::InvalidData(s)` where `s` describes a shape failure ("entry data is empty", "components is 0", "parse U16Array error", "parse U32Array error") | `EntryError::InvalidShape { format, count }` | These are shape-level failures dressed up as InvalidData. |

**Static-string table for `InvalidValue`:**

| Old payload | New `&'static str` |
|-------------|--------------------|
| `format!("invalid time format: {value}")` (chrono ParseError) | `"invalid time format"` |
| `"invalid gps offset"` | `"invalid gps offset"` |
| Any "non utf-8" / FromUtf8Error wrap | `"invalid utf-8"` |
| GPS ref char not in `N/S/E/W` (currently does NOT error — see P5 — but if it did) | `"invalid gps direction"` (reserve string; not used yet) |
