# nom-exif v3 — P4.5: drop PartialVec, unify on bytes::Bytes + drop pooled buf

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Eliminate the internal `PartialVec` / `AssociatedInput` types entirely, unifying every byte-view in the parser on `bytes::Bytes`. Replace the multi-slot `Buffers` pool with a single-slot `Option<Bytes>` cache + `try_into_mut` recycle. No public API changes — all `pub(crate)` plumbing.

**Architecture:** `Bytes` becomes the sole byte-view type. `IfdIter::input` and `ExifIter::input` hold `Bytes` directly (no `Arc<PartialVec>` wrap — `Bytes::clone` is already a refcount bump). The CR3 multi-block path is restructured to *upstream-once slicing*: `parse_cr3_exif_iter` slices every CMT block into its own `Bytes` at the moment it has the full parser buffer in hand, instead of letting `ExifIter` reach back through state to do it lazily. `MediaParser`'s buffer subsystem is reduced to a single `Option<Bytes>` cache: acquire tries `Bytes::try_into_mut` (succeeds iff the previous `ExifIter` is gone, refcount==1, zero-copy reuse); otherwise allocates fresh.

**Tech Stack:** `bytes` 1.7.1 (already a hard dependency in `Cargo.toml`), Rust 1.83.

**Phase position:** lands on `v3` branch between P4 (values) and P5 (exif-iter). Master plan (`docs/superpowers/plans/2026-05-08-v3-master.md`) phase summary table updated as the last task to add a P4.5 row and a P7 placeholder (memory source, separate plan).

**Why now:** P5 will heavily edit `ExifIter` / `Exif` and the byte-view plumbing they sit on. Doing the unification *before* P5 keeps P5's diff focused on API ergonomics rather than mixed concerns. Also unblocks P7 (`MediaSource::from_bytes(Bytes)` zero-copy memory source) since the streaming and memory parse paths can both terminate in `Bytes::clone`.

**Why path B (delete PartialVec) over path A (just swap its internals):** see `docs/V3_API_DESIGN.md` §8.10. Brief: only one of the four `partial(&[u8])` callsites genuinely needs "reach beyond current view" semantics (CR3), and that callsite is best solved by upstream slicing. Carrying a `PartialVec` shell around an otherwise-pure `Bytes` interface is residual abstraction; v3's job is to converge.

**Exit criterion:**
- `src/partial_vec.rs` deleted; no `PartialVec` / `AssociatedInput` symbol anywhere.
- `IfdIter::input: Bytes` and `ExifIter::input: Bytes` (no `Arc` wrap).
- `TiffDataBlock { block_id: String, data: Bytes, header: Option<TiffHeader> }` (range field replaced with pre-sliced `Bytes`).
- `add_tiff_block(block_id: String, data: Bytes, header: Option<TiffHeader>)` — signature change.
- `ShareBuf::share_buf(&mut self) -> (Bytes, usize)` — returns full buffer + position offset; caller does the slicing.
- `parse_cr3_exif_iter` slices all CMT blocks upfront.
- `src/buffer.rs` deleted; `MediaParser::new()` zero-alloc.
- `cargo test --all-features` green.
- `cargo doc --no-deps --all-features --document-private-items` clean.
- `cargo clippy --all-features -- -D warnings` clean.
- Public symbols in `src/lib.rs::pub use` unchanged from before P4.5.

```rust
    pub(crate) data: Bytes,
    pub(crate) range: Range<usize>,
}

impl Debug for PartialVec {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        f.debug_struct("PartialVec")
            .field("data len", &self.data.len())
            .field("range", &self.range)
            .finish()
    }
}

impl PartialVec {
    pub(crate) fn new(data: Bytes, range: Range<usize>) -> PartialVec {
        assert!(range.end <= data.len());
        PartialVec { data, range }
    }

    pub(crate) fn from_vec(vec: Vec<u8>) -> PartialVec {
        let range = 0..vec.len();
        PartialVec {
            data: Bytes::from(vec),
            range,
        }
    }

    pub(crate) fn from_vec_range(vec: Vec<u8>, range: Range<usize>) -> PartialVec {
        assert!(range.end <= vec.len());
        PartialVec {
            data: Bytes::from(vec),
            range,
        }
    }

    pub(crate) fn partial(&self, subslice: &[u8]) -> AssociatedInput {
        let range = self
            .data
            .subslice_in_range(subslice)
            .expect("subslice should be a sub slice of self");
        PartialVec {
            data: self.data.clone(),
            range,
        }
    }

    /// Convert this view into a standalone `Bytes` (the sliced view, not
    /// the full backing allocation). Used during the P4.5 migration to
    /// hand off byte ownership to `Bytes`-typed consumers.
    pub(crate) fn into_bytes(self) -> Bytes {
        self.data.slice(self.range)
    }
}

impl From<Vec<u8>> for PartialVec {
    fn from(value: Vec<u8>) -> Self {
        PartialVec::from_vec(value)
    }
}

impl From<(Vec<u8>, Range<usize>)> for PartialVec {
    fn from(value: (Vec<u8>, Range<usize>)) -> Self {
        let (data, range) = value;
        PartialVec::from_vec_range(data, range)
    }
}

impl Deref for PartialVec {
    type Target = [u8];
    fn deref(&self) -> &Self::Target {
        &self.data[self.range.clone()]
    }
}

impl AsRef<[u8]> for PartialVec {
    fn as_ref(&self) -> &[u8] {
        &self.data[self.range.clone()]
    }
}

impl Borrow<[u8]> for PartialVec {
    fn borrow(&self) -> &[u8] {
        &self.data[self.range.clone()]
    }
}
```

```rust
const MAX_REUSE_BUF_SIZE: usize = 1024 * 1024;

/// Buffer-management state used by `MediaParser` (sync and async paths share it).
///
/// Holds at most one *active* `Vec<u8>` (being filled by the current parse) and
/// one *cached* `Bytes` clone of the most recently shared buffer. When the
/// next parse starts, the cache is consulted: if `Bytes::try_into_mut`
/// succeeds the underlying allocation is reused (the previous `ExifIter`
/// has been dropped); otherwise the clone is discarded and a fresh
/// `Vec<u8>` is allocated.
///
/// This replaces the v2 multi-slot `Buffers` pool — `MediaParser` methods
/// are `&mut self`, so a single slot is sufficient.
#[derive(Debug, Default)]
pub(crate) struct BufferedParserState {
    cached: Option<bytes::Bytes>,
    buf: Option<Vec<u8>>,
    position: usize,
}

impl BufferedParserState {
    pub(crate) fn new() -> Self {
        Self::default()
    }

    pub(crate) fn reset(&mut self) {
        // If a parse failed mid-way the buf may still be present; drop it.
        // Cache stays — recycle on next acquire if eligible.
        self.buf = None;
        self.position = 0;
    }

    pub(crate) fn acquire_buf(&mut self) {
        debug_assert!(self.buf.is_none());
        let buf = match self.cached.take() {
            Some(b) => match b.try_into_mut() {
                Ok(bm) => {
                    let mut v = Vec::<u8>::from(bm);
                    v.clear();
                    if v.capacity() > MAX_REUSE_BUF_SIZE {
                        v.shrink_to(MAX_REUSE_BUF_SIZE);
                    }
                    v
                }
                Err(_still_shared) => Vec::with_capacity(INIT_BUF_SIZE),
            },
            None => Vec::with_capacity(INIT_BUF_SIZE),
        };
        self.buf = Some(buf);
    }

    pub(crate) fn buf(&self) -> &Vec<u8> {
        self.buf.as_ref().expect("no buf here")
    }

    pub(crate) fn buf_mut(&mut self) -> &mut Vec<u8> {
        self.buf.as_mut().expect("no buf here")
    }

    #[cfg(test)]
    pub(crate) fn cached_ptr_for_test(&self) -> Option<*const u8> {
        self.cached.as_ref().map(|b| b.as_ptr())
    }

    #[cfg(test)]
    pub(crate) fn buf_is_none_for_test(&self) -> bool {
        self.buf.is_none()
    }
}

impl Buf for BufferedParserState {
    fn buffer(&self) -> &[u8] {
        &self.buf()[self.position..]
    }
    fn clear(&mut self) {
        self.buf_mut().clear();
    }
    fn set_position(&mut self, pos: usize) {
        self.position = pos;
    }
    fn position(&self) -> usize {
        self.position
    }
}

impl ShareBuf for BufferedParserState {
    fn share_buf(&mut self) -> (bytes::Bytes, usize) {
        let vec = self.buf.take().expect("no buf to share");
        let bytes = bytes::Bytes::from(vec);
        let position = self.position;
        self.cached = Some(bytes.clone());
        (bytes, position)
    }
}
```
