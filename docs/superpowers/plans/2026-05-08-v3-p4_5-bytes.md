# nom-exif v3 — P4.5: PartialVec → bytes::Bytes + drop pooled buf

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the hand-rolled `Arc<Vec<u8>>` storage in `PartialVec` with `bytes::Bytes`, and replace the multi-slot `Buffers` pool with a single-slot `Option<Bytes>` cache + `try_into_mut` recycle. No public API changes; all internals.

**Architecture:** `PartialVec` keeps its name and shape (`{ data, range }`) but `data: Arc<Vec<u8>>` becomes `data: Bytes` (whole-alloc reference). `MediaParser`'s buffer state drops the `Buffers` struct (`shared`/`pool`/`acquired` 三态) and holds `cached: Option<Bytes>` only — recycle is done by `Bytes::try_into_mut` which succeeds iff the alloc is uniquely owned (i.e., the previously-shared `ExifIter` has been dropped). Sync and async fill paths unchanged: they still fill into `Vec<u8>`, then `Bytes::from(vec)` at share-time (zero-copy in `bytes` 1.x).

**Tech Stack:** `bytes` 1.7.1 (already a hard dependency in `Cargo.toml`), Rust 1.83.

**Phase position:** lands on `v3` branch between P4 (values) and P5 (exif-iter). Master plan (`docs/superpowers/plans/2026-05-08-v3-master.md`) phase summary table will be updated as the last task to add a P4.5 row and a P7 row (memory source, separate plan).

**Why now:** P5 will heavily edit `ExifIter` / `Exif` and the `PartialVec` plumbing they sit on. Doing the storage swap *before* P5 keeps P5's diff focused on API ergonomics rather than mixed concerns. Also unblocks a future P7 (`MediaSource::from_bytes(Bytes)` zero-copy memory source) by giving us native `Bytes` types throughout.

**Exit criterion:**
- `PartialVec.data: Bytes` (not `Arc<Vec<u8>>`).
- `src/buffer.rs` deleted.
- `MediaParser::new()` performs no upfront allocation (was 2 × 4 KB).
- `cargo test --all-features` green.
- `cargo doc --no-deps --all-features --document-private-items` clean.
- `cargo clippy --all-features -- -D warnings` clean.

**Why keep `PartialVec` as a struct rather than collapse to `Bytes`:** the CR3 path (`exif_iter.rs:401`) needs to construct a *new* view (different range) over the *full* parser buffer. With raw `Bytes`, slicing is one-way — once you have a sliced `Bytes` you cannot recover the parent. Keeping `PartialVec { full_data: Bytes, view_range }` preserves this affordance with a one-line change. Renaming `PartialVec` → `BytesView` is an optional polish for v3 follow-up; not in this plan.

use crate::slice::SubsliceRange as _;

use std::borrow::Borrow;
use std::fmt::Debug;
use std::ops::Deref;
use std::ops::Range;

use bytes::Bytes;

/// Owning view into a shared byte buffer.
///
/// `data` is a `bytes::Bytes` that references the entire underlying
/// allocation; `range` selects the currently-visible window. CR3 and
/// other multi-block formats reach back into `data` to construct sibling
/// views over the same allocation, which is why we keep the full `Bytes`
/// rather than a pre-sliced one.
#[derive(Clone, PartialEq, Eq, Default)]
pub(crate) struct PartialVec {
    pub(crate) data: Bytes,
    pub(crate) range: Range<usize>,
}

impl Debug for PartialVec {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        f.debug_struct("PartialVec")
            .field("data len", &self.data.len())
            .field("range", &self.range)
            .finish()
    }
}

impl PartialVec {
    pub(crate) fn new(data: Bytes, range: Range<usize>) -> PartialVec {
        assert!(range.end <= data.len());
        PartialVec { data, range }
    }

    pub(crate) fn from_vec(vec: Vec<u8>) -> PartialVec {
        let range = 0..vec.len();
        PartialVec {
            data: Bytes::from(vec),
            range,
        }
    }

    pub(crate) fn from_vec_range(vec: Vec<u8>, range: Range<usize>) -> PartialVec {
        assert!(range.end <= vec.len());
        PartialVec {
            data: Bytes::from(vec),
            range,
        }
    }

    /// Construct a sibling view (over the same underlying allocation) for
    /// `subslice`. Panics if `subslice` is not within `self.data`.
    pub(crate) fn partial(&self, subslice: &[u8]) -> AssociatedInput {
        let range = self
            .data
            .subslice_in_range(subslice)
            .expect("subslice should be a sub slice of self");
        PartialVec {
            data: self.data.clone(),
            range,
        }
    }
}

impl From<Vec<u8>> for PartialVec {
    fn from(value: Vec<u8>) -> Self {
        PartialVec::from_vec(value)
    }
}

impl From<(Vec<u8>, Range<usize>)> for PartialVec {
    fn from(value: (Vec<u8>, Range<usize>)) -> Self {
        let (data, range) = value;
        PartialVec::from_vec_range(data, range)
    }
}

impl Deref for PartialVec {
    type Target = [u8];
    fn deref(&self) -> &Self::Target {
        &self.data[self.range.clone()]
    }
}

impl AsRef<[u8]> for PartialVec {
    fn as_ref(&self) -> &[u8] {
        &self.data[self.range.clone()]
    }
}

impl Borrow<[u8]> for PartialVec {
    fn borrow(&self) -> &[u8] {
        &self.data[self.range.clone()]
    }
}

pub(crate) type AssociatedInput = PartialVec;

const MAX_REUSE_BUF_SIZE: usize = 1024 * 1024;

/// Buffer-management state used by `MediaParser` (sync and async paths share it).
///
/// Holds at most one *active* `Vec<u8>` (being filled by the current parse) and
/// one *cached* `Bytes` clone of the most recently shared buffer. When the next
/// parse starts, the cache is consulted: if its `try_into_mut` succeeds the
/// underlying allocation is reused (the previous `ExifIter` has been dropped);
/// otherwise we drop the clone and allocate a fresh `Vec<u8>`.
///
/// This replaces the multi-slot `Buffers` pool from v2 — a single slot is
/// sufficient because `MediaParser` methods are `&mut self` (single-threaded
/// per parser instance).
#[derive(Debug, Default)]
pub(crate) struct BufferedParserState {
    cached: Option<bytes::Bytes>,
    buf: Option<Vec<u8>>,
    position: usize,
}

impl BufferedParserState {
    pub(crate) fn new() -> Self {
        Self::default()
    }

    pub(crate) fn reset(&mut self) {
        // If a parse failed mid-way we may still hold an unfilled-or-partial
        // Vec; drop it. Cache stays — it'll either be recycled on next acquire
        // or replaced.
        self.buf = None;
        self.position = 0;
    }

    pub(crate) fn acquire_buf(&mut self) {
        debug_assert!(self.buf.is_none());
        let buf = match self.cached.take() {
            Some(b) => match b.try_into_mut() {
                Ok(bm) => {
                    let mut v = Vec::<u8>::from(bm);
                    v.clear();
                    if v.capacity() > MAX_REUSE_BUF_SIZE {
                        v.shrink_to(MAX_REUSE_BUF_SIZE);
                    }
                    v
                }
                Err(_still_shared) => Vec::with_capacity(INIT_BUF_SIZE),
            },
            None => Vec::with_capacity(INIT_BUF_SIZE),
        };
        self.buf = Some(buf);
    }

    pub(crate) fn buf(&self) -> &Vec<u8> {
        self.buf.as_ref().expect("no buf here")
    }

    pub(crate) fn buf_mut(&mut self) -> &mut Vec<u8> {
        self.buf.as_mut().expect("no buf here")
    }

    #[cfg(test)]
    pub(crate) fn cached_ptr_for_test(&self) -> Option<*const u8> {
        self.cached.as_ref().map(|b| b.as_ptr())
    }

    #[cfg(test)]
    pub(crate) fn buf_is_none_for_test(&self) -> bool {
        self.buf.is_none()
    }
}

impl Buf for BufferedParserState {
    fn buffer(&self) -> &[u8] {
        &self.buf()[self.position..]
    }
    fn clear(&mut self) {
        self.buf_mut().clear();
    }
    fn set_position(&mut self, pos: usize) {
        self.position = pos;
    }
    fn position(&self) -> usize {
        self.position
    }
}

impl ShareBuf for BufferedParserState {
    fn share_buf(&mut self, mut range: Range<usize>) -> PartialVec {
        let vec = self.buf.take().expect("no buf to share");
        range.start += self.position;
        range.end += self.position;
        let bytes = bytes::Bytes::from(vec);
        self.cached = Some(bytes.clone());
        PartialVec::new(bytes, range)
    }
}
