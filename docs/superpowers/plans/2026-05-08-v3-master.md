# nom-exif v3 — Master Implementation Plan

> **For agentic workers:** this is the **index** for the v3 cutover. The actual task lists live in the per-phase plans linked below. Use superpowers:subagent-driven-development to execute each phase plan task-by-task.

**Goal:** Cut the public API of `nom-exif` from v2.x (~16 surface symbols, internals leaking) to the v3 design (~29 deliberate symbols, structured errors, single `MediaParser`, ergonomic `read_exif`).

**Spec:** [`docs/V3_API_DESIGN.md`](../../V3_API_DESIGN.md). Decisions there are final unless renegotiated in writing.

**Branch strategy:** all phases land sequentially on `v3` (the current branch). No per-phase feature branches. Each phase ends with `cargo test --all-features` green and a tagged commit (`v3-p1-done`, etc.) so we can revert to a phase boundary if needed.

**Tech stack:** Rust 1.80 → 1.83 MSRV bump, `thiserror` 2.0, `chrono` 0.4, `nom` 8.0, `tokio` 1.40 (feature-gated).

## Cross-phase rules

- **MSRV bump to 1.83+** lands in **P1** (needed for `Option::is_some_and`, `expect` lint policy). Update `Cargo.toml::package.rust-version` and CI matrix in P1 task 0.
- **Public symbols only added in their owning phase.** Don't preview future phase types under `#[doc(hidden)]` — the migration guide gets confused.
- **Deletions are the **last** task of each phase**, after all internal call sites have been migrated. This keeps the build green between tasks.
- **No `#[deprecated]` shims.** Per `AGENTS.md` "no backwards-compatibility hacks" — v3 is a clean break, not a graceful fade.
- **Test discipline.** Every renamed/added public method gets a unit test in P[X], not deferred. Snapshot tests against fixtures in `testdata/` are preferred over inline byte literals.
- **Doc tests count as breakage.** Run `cargo test --all-features --doc` after each phase. The `lib.rs` doc-comment in v2 has runnable examples — they break in P3 and only get rewritten in P6. Strategy: in P3, gate the v2 doc-comment behind `#[cfg(any())]` (i.e. dead-code-comment it) and write a *minimal* placeholder; full rewrite is P6.

## Risk register

| Risk | Phase | Mitigation |
|------|-------|------------|
| Internal `ParseEntryError::InvalidData(String)` sites need rephrasing as `EntryError::InvalidValue(&'static str)` | P1 | Plan task explicitly enumerates each site + the static string to use. |
| Removing `MediaSource::tcp_stream` breaks `examples/rexiftool` if it uses it | P3 | Grep first, verify only `file_path` is used. |
| Async/sync unification (P2) is hard to hold in head while keeping public API stable | P2 | Plan factors out the shared trait first, then migrates sync, then async — each task compiles. |
| `Exif::iter()` lifetime over internal `Vec<ParsedImageFileDirectory>` may need GATs / explicit lifetime | P5 | Prototype in a scratch task first; design currently uses `impl Iterator<Item = ExifEntry<'_>>` which is fine on Rust 1.80+. |
| Doc examples in `lib.rs` rotting between phases | All | Master plan commitment: `lib.rs` `//!` examples are minimized in P3 and fully rewritten in P6. |

## Done definition (whole v3)

1. Every public symbol in §4.1 of the spec exists with the documented signature.
2. Every example in §5 (migration guide) compiles and runs (verified by `cargo test --doc` against migration-guide doc-tests added in P6).
3. `cargo doc --no-deps --all-features --document-private-items` produces no warnings.
4. `cargo clippy --all-features -- -D warnings` clean.
5. Fuzz harness in `fuzz/` still builds.
6. CHANGELOG.md has a `## [3.0.0]` section with the §5 migration table.
7. README.md examples use the v3 surface (no `MediaSource::file_path`, no `parser.parse::<_,_,ExifIter>`).
