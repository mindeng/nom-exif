# PNG Support — Master Implementation Plan

> **For agentic workers:** this is the **index** for the v3.3 PNG-support PR. The actual task lists live in the per-phase plans linked below. Use superpowers:subagent-driven-development to execute each phase plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add PNG support (standard `eXIf`, `tEXt` chunks, legacy hex-encoded EXIF) to nom-exif, behind a new `parse_image_metadata` entry point that returns a structured `ImageMetadata { exif, format }`. Bundle the source-input model unification (`MediaSource::from_memory` + deprecation of `from_bytes` family) so the new API ships consistent with the rest of the crate.

**Spec:** [`docs/superpowers/specs/2026-05-10-png-support-design.md`](../specs/2026-05-10-png-support-design.md). Decisions there are final unless renegotiated in writing.

**Branch strategy:** all phases land sequentially on `main`. Single PR, seven commits. Each phase ends with `cargo test --all-features` green.

**Tech stack:** Rust (existing MSRV), `bytes` crate, `nom`, `tokio` (feature-gated). No new dependencies.

## Cross-phase rules

- **CI gate**: every phase ends with `cargo test --all-features` green AND `cargo fmt --check` clean (per `feedback_cargo_fmt_before_commit.md` — fmt is a hard CI gate).
- **No premature exports**: public symbols only ship in the phase that owns them (P3 owns the new types; P0 owns `from_memory`).
- **Deprecation warnings**: deprecated symbols intentionally still compile and pass their tests. In-crate test files that exercise deprecated paths get `#[allow(deprecated)]`. CI must not treat warnings as errors (verify in P0 task 0).
- **Atomic commits**: each task ends with a commit. Commit messages follow existing repo style (lower-case `feat:` / `fix:` / `docs:` / `refactor:` / `test:` prefix; concise body).
- **Test discipline**: every public method gets a unit test in the phase that introduces it, not deferred. Inline byte literals OK for synthetic PNG fixtures (CRC unverified, see spec).
- **Single PR**: all phases land on `main`; the seven commits become the PR's commit history. No squash-merge — preserve phase structure.

## Done definition (whole PR)

1. `read_exif("foo.png")` returns `Ok(Exif)` for any of: pure `eXIf`, legacy `Raw profile type exif`, legacy `Raw profile type APP1`, mixed `eXIf+legacy` (eXIf wins).
2. `parser.parse_image_metadata(MediaSource::open("foo.png")?)` returns `ImageMetadata { exif: Some(_), format: Some(FormatMetadata::Png(_)) }` when both EXIF and `tEXt` are present.
3. PNG with only `tEXt` (no EXIF): `parse_exif` returns `Err(ExifNotFound)`; `parse_image_metadata` returns `Ok(ImageMetadata { exif: None, format: Some(_) })`.
4. `MediaSource::from_memory(bytes)` accepts every type accepted by today's `MediaSource::from_bytes` (zero-copy preserved verbatim — verify via pointer-identity test).
5. `parse_exif(MediaSource::from_memory(bytes))` produces byte-identical EXIF compared to `parse_exif_from_bytes(MediaSource::from_bytes(bytes))` on the same input.
6. All deprecated symbols still compile and pass their original tests (with `#[allow(deprecated)]` annotations where needed).
7. `cargo test --all-features` green; `cargo fmt --check` clean; `cargo doc --no-deps --all-features` produces no new warnings.
8. CHANGELOG documents Added / Deprecated / Notes for v3.3.0.
9. README + lib.rs module docs use the v3.3 surface (no `from_bytes` in non-deprecation contexts).
10. PR diff stays focused: only `src/png.rs`, `src/image_metadata.rs`, `src/exif/png_text.rs`, edits to `src/file.rs` / `src/exif.rs` / `src/parser.rs` / `src/parser_async.rs` / `src/lib.rs`, plus tests / fixtures / docs / CHANGELOG. No collateral refactors.
